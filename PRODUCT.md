# Product

## Register

product

## Users

ENCORE is used by ticket buyers, check-in staff, venue operators, and administrators. The admin surface is used in a focused operations context: checking show readiness, schedule status, ticket inventory, order state, refunds, and staff access.

## Product Purpose

ENCORE manages the full ticketing loop from show publishing to seat selection, order payment, electronic tickets, check-in, and admin reporting. Success means operators can see current state quickly, make status changes confidently, and trust that frontend state reflects backend results after each operation.

## Brand Personality

Refined, direct, controlled. The user-facing side can carry ENCORE's black-red premium ticketing mood; the admin side should feel quieter and more operational, with restrained color used for actions and state.

## Anti-references

Avoid marketing-style admin pages, decorative card piles, oversized rounded panels, glassy effects used only for style, red-blue glow backgrounds, and redundant headings that repeat the shell navigation.

## Design Principles

- Task first: layout density should help scanning, filtering, and acting on records.
- State is evidence: every status badge, count, and row action should map to backend data and update from server responses.
- One shell, many workflows: the admin layout owns navigation and page context; child pages should avoid duplicate hero headers.
- Less container noise: use rows, dividers, and compact toolbars before adding cards.
- Familiar controls: Element Plus tables, forms, dialogs, and buttons should stay predictable and consistent.

## Accessibility & Inclusion

Target readable contrast on dark surfaces, keyboard-accessible standard controls, reduced decorative motion, and responsive layouts that avoid horizontal overflow on tablet and small desktop widths.
