## 关键实现

- 项目记忆采用仓库文件方案，不依赖平台级跨对话记忆。
- 后续每次开发前默认读取 `PROJECT_MEMORY.md`、`DEVELOPMENT_PLAN.md` 和最新日志。
- 后续每完成一个可运行闭环，执行验证、更新日志、提交并推送到 `origin/main`。
- 后端第一批 API 保持 `/api/**` 前缀，返回统一结构，便于前端 Axios 后续统一拦截。
- Sa-Token 当前保留 Redis 集成，后续登录态可直接扩展到分布式部署。
- Docker Compose 只提交服务定义和初始化脚本，不提交本地 `.env`、数据卷或运行日志。
- 前端 token header 名称和值由后端登录响应决定，避免把 token 名称写死在业务页面里。
- 前端 `.env.example` 只记录公开 API 地址变量；真实生产地址后续在云服务器环境中配置。
- 语言切换控件使用固定宽度分段按钮，避免中英文文本长度差异导致导航抖动。
- 锁座采用 Redis TTL，数据库只保存最终 SOLD/DISABLED 等持久状态，查询座位图时叠加 Redis 锁状态返回 LOCKED。
- 创建订单时以后端数据库座位价格重新计算金额，不信任前端传入金额。
- 同一用户、同一场次、同一组座位重复创建订单时复用现有待支付订单，避免重复提交生成多笔订单。

## 问题与解决

- 问题：此前规划中认为远程仓库尚未提供，但实际检查发现 GitHub remote 已经配置。
- 解决：直接沿用当前 `origin`，不重新初始化仓库，也不改变已有提交历史。
- 问题：新对话无法保证自动携带长期上下文。
- 解决：将上下文写入 `PROJECT_MEMORY.md`，并约定新会话先读取该文件。
- 问题：Docker Desktop 起初未运行，`docker compose up -d mysql redis` 无法连接 Docker API。
- 解决：Docker Desktop 启动后重新执行 Compose，MySQL/Redis 均健康。
- 问题：本机 3306 存在 MySQL 服务，但默认 `encore/encore123` 和 `root/root123` 无法登录。
- 解决：未擅自修改本机数据库权限，将项目 MySQL 宿主机映射端口改为 `3307`，后端 dev 默认端口同步改为 `3307`。
- 问题：GitHub HTTPS 推送一度因凭据/交互流程卡住并超时。
- 解决：清理卡住的 Git 进程后确认前 4 个本地提交已同步到 `origin/main`。
