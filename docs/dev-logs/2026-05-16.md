## 完成内容

- 审阅当前未提交改动，确认主要集中在登录页、首页、个人页、后台场次/订单页和多语言文案。
- 使用 UI/UX 检查约束重新审视当前黑金票务界面，保留现有风格并避免破坏用户已做的页面主体。
- 登录页语言切换改为调用统一 `setAppLocale`，继续持久化语言偏好并同步 `html lang` / `data-locale`。
- 前端 Axios `requestData` 增强了错误解包，后端返回非 2xx 且包含统一响应体时会抛出业务消息。
- 恢复订单、支付、电子票、座位、后台演出管理、语言切换等页面仍在使用的中英文 i18n 键。
- 新增后端 DTO：`CheckInRequest`、`CheckInResponse`。
- 新增后端 `CheckInController`，提供 `POST /api/checkin/verify`。
- 新增后端 `CheckInService`，按票码查询票项并执行一次性核销。
- 检票接口限制 `checker`、`admin`、`sysadmin` 角色使用。
- 有效票据要求订单为 `PAID` 且票状态为 `UNUSED`，核销后更新为 `CHECKED_IN`。
- 重复核销、未支付/失效订单、无效票码、非 UNUSED 状态都会返回业务错误。
- 新增前端 `api/checkin.ts`。
- 检票页从本地 `XYZ` 后缀模拟逻辑切换为调用后端检票接口，并显示核销成功的演出、剧场和座位信息。
- 新增 `CheckInServiceTest`，覆盖成功核销、重复核销拒绝、普通用户无权限。

## 关键实现

- 票码仍沿用 Phase 3 支付成功后生成的 `ticket_item.ticket_code`。
- 当前数据库表未新增 `checked_in_at` 字段，核销时间通过 `ticket_item.updated_at` 和接口响应中的 `checkedInAt` 表达。
- 检票服务优先校验角色，避免普通用户通过接口试探票码。
- 检票页保留极简大屏扫描视觉，但增加明确的提交按钮和加载状态，键盘回车仍可核销。

## 问题与解决

- 问题：本轮 UI 改动删除了多个仍在使用的 i18n 键，构建不会失败但运行时会显示缺失文案。
- 解决：恢复所有核心演示页面仍引用的中英文键，避免订单/支付/电子票/后台页面出现键名。
- 问题：本轮前半段 Docker Desktop 未运行，`docker compose ps` 无法连接 Docker API，8080 后端也未处于可用状态。
- 解决：Docker Desktop 恢复后启动 MySQL/Redis 和 Spring Boot 后端，完成真实购票与检票链路实调。
- 问题：MyBatis-Plus `updateById` 在测试中存在单对象和集合重载，Mockito 断言参数类型不明确。
- 解决：在测试里显式使用 `TicketItem` 类型匹配器。
